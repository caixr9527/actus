# Actus 产品想法 · PRD Idea

> 基于对现有代码的分析，记录各页面的现状、UX 问题与改进方向。
> 更新日期：2026-03-04

---

## 1. 首页（Home Page）

**文件**: `src/app/page.tsx`, `src/components/suggested-questions.tsx`

### 现状

- 居中布局：欢迎语 + 聊天输入框 + 推荐问题列表
- 推荐问题从 `src/config/app.config.ts` 中读取静态数组
- 垂直居中通过 `-mt-12 sm:-mt-16` 负 margin 补偿 header 高度实现
- 发送后创建会话并跳转到详情页，初始消息通过 URL 参数传递

### 问题 / 痛点

- **推荐问题静态化**：写死在配置文件中，无法反映真实使用场景，用户用完一次后就无新鲜感
- **负 margin 不稳定**：`-mt-12/-mt-16` 是经验值，resize 或 header 高度变化时视觉居中会偏移
- **缺乏任务上下文入口**：首页只有新建入口，没有"最近任务"等快速入口，老用户每次都要去侧边栏翻历史
- **输入框无草稿缓存**：刷新页面后输入内容丢失

### 改进想法

- **动态推荐问题**：接入后端 API，按类别（搜索/代码/文件处理）或历史任务频次动态生成推荐
- **近期任务快捷入口**：在推荐问题下方展示最近 3-5 条历史任务，一键续接
- **任务模板库**：提供预设任务模板（如"分析这份 PDF"、"爬取并汇总网页"），降低新用户的上手门槛
- **草稿本地缓存**：用 `localStorage` 暂存未发送的输入内容，刷新后恢复

## 2. 会话详情页（Session Detail）

**文件**: `src/components/session-detail-view.tsx`

### 现状

- 主区域：Timeline 消息流 + 底部 PlanPanel + ChatInput
- 右侧：文件预览面板 / 工具预览面板（宽度固定 600px）
- 任务运行中自动追踪最新工具预览
- 支持 VNC 全屏覆盖层查看远程桌面
- 工具可点击但无任何 hover 视觉提示（只在 `handleToolClick` 中做 kind 过滤）

### 问题 / 痛点

- **工具卡片无可点击提示**：用户不知道工具卡片可以点击，容易错过侧边预览功能
- **Plan 无历史回溯**：`getLatestPlanFromEvents` 只取最新 Plan，重新规划后旧计划丢失，无法回看
- **预览面板宽度固定**：`w-[600px]` 硬编码，小屏上挤压主内容区，大屏上留空过多
- **长会话无虚拟化**：Timeline 全量渲染，百条事件时页面卡顿
- **停止任务无确认**：点击暂停按钮直接调 API，无二次确认，容易误操作
- **错误态样式简陋**：错误时只有红字 + 重试链接，无状态码、时间、排查建议

### 改进想法

- **工具卡片交互提示**：鼠标 hover 时显示 `cursor-pointer` + 淡入 tooltip（"点击查看详情"）
- **Plan 历史记录**：保留所有历史 Plan 版本，PlanPanel 顶部加时间轴或版本切换器
- **可拖拽预览宽度**：在预览面板左边缘添加拖拽 resize 手柄，并记忆用户偏好
- **Timeline 虚拟滚动**：超过 50 条 item 时启用虚拟列表（`react-window` 或自实现）
- **停止任务二次确认**：点击暂停后弹出 Popover 确认，防止误操作

## 3. 左侧边栏（Left Panel）

**文件**: `src/components/left-panel.tsx`, `src/components/session-list.tsx`

### 现状

- 侧边栏顶部：折叠/展开按钮
- 新建任务按钮（⌘K 快捷键提示）
- 会话列表（`SessionList` 组件）

### 问题 / 痛点

- **无搜索功能**：历史任务多时无法快速定位，只能逐条滚动
- **无状态筛选**：无法快速过滤"运行中"、"等待"、"已完成"任务
- **无分组**：所有任务平铺，没有按日期、主题等维度分组
- **无置顶/收藏**：重要任务无法标记，每次都要翻找
- **⌘K 快捷键提示只是装饰**：KbdGroup 显示了快捷键但代码中未实际注册该快捷键

### 改进想法

- **会话搜索框**：侧边栏顶部增加搜索输入，实时过滤会话标题
- **状态筛选 Tab**：全部 / 运行中 / 待处理 / 已完成
- **按日期分组**：今天、昨天、过去 7 天、更早
- **置顶 / 收藏**：右键菜单或长按出现操作菜单
- **实际注册 ⌘K 快捷键**：在根布局用 `useEffect` 监听 `keydown` 事件，触发新建任务并聚焦输入框

## 4. 聊天输入框（Chat Input）

**文件**: `src/components/chat-input.tsx`

### 现状

- 支持多行文本输入（Ctrl/Cmd+Enter 发送）
- 支持文件上传（多文件并发上传，展示文件名+大小+删除）
- 发送中显示 loading；任务运行中显示暂停按钮
- 文件上传无类型/大小限制（`<input type="file" multiple>`）

### 问题 / 痛点

- **无文件限制提示**：用户不知道允许哪些文件类型、最大多少 MB
- **上传无逐文件进度**：多文件上传时整体只有一个 Loader，不知道哪个文件在上传
- **暂停按钮无 tooltip**：图标含义不够直观，初次使用的用户可能不知道这是"停止任务"
- **textarea 高度固定**：`rows={2}` + `h-[46px]` 固定高度，长提示词时无法自动扩展
- **粘贴图片无支持**：常见的截图粘贴场景未支持，需要先保存再上传
- **无拖拽上传区域**：文件拖入浏览器时没有视觉反馈和上传支持

### 改进想法

- **文件限制说明**：上传按钮 tooltip 标注支持类型和大小上限
- **粘贴图片直传**：监听 `paste` 事件，检测 `clipboardData.files`，自动触发上传
- **拖拽上传**：整个输入框区域支持文件拖入，拖入时高亮边框提示
- **textarea 自动扩展高度**：使用 `scrollHeight` 动态调整，设置 `min-h` 和 `max-h` 避免无限增长
- **暂停按钮 tooltip**：悬停时显示"停止当前任务"

## 5. 工具调用预览（Tool Preview Panel）

**文件**: `src/components/tool-preview-panel.tsx`

### 现状

- 支持 bash / browser / search / file / mcp / a2a / default 七种预览模式
- bash 和 file 使用暗色终端风格展示
- browser 展示截图 + URL + VNC 入口
- search 展示结果列表，支持点击跳转
- 全部使用 `<pre>` 展示原始文本，无语法高亮

### 问题 / 痛点

- **代码无语法高亮**：bash 输出、file 内容、JSON 结果均使用 `<pre>` 纯文本，可读性差
- **无复制按钮**：bash 命令输出、文件内容无法一键复制
- **浏览器截图无时效提示**：截图会自动更新（通过 resolvedPreviewTool 机制），但视觉上看不出更新时间
- **无工具执行耗时**：不知道这个工具调用花了多少时间
- **面板标题固定为"Actus 的电脑"**：对 MCP/A2A 工具语义不准确
- **搜索结果无 favicon**：搜索结果列表只有文字，缺少网站图标

### 改进想法

- **代码语法高亮**：引入 `shiki` 或 `prism-react-renderer`，对 bash/file/JSON 内容做高亮
- **一键复制**：在 bash、file、mcp、a2a 面板的右上角添加复制按钮
- **截图时间戳**：在浏览器截图右下角叠加更新时间（格式：`HH:mm:ss`）
- **工具执行耗时**：在 header 区域展示工具从调用到返回的耗时（需后端提供时间戳）
- **搜索结果 favicon**：使用 `https://www.google.com/s2/favicons?domain=` 加载网站图标

## 6. 任务计划面板（Plan Panel）

**文件**: `src/components/plan-panel.tsx`

### 现状

- 折叠/展开两态切换
- 折叠态：显示 `steps[0]` 描述 + `completedCount / totalCount`
- 展开态：完整步骤列表，已完成步骤显示 Check 图标，其余显示 Clock 图标
- 步骤只显示文字，不可交互，描述文字有 `truncate` 截断

### 问题 / 痛点

- **折叠态展示第一个步骤而非当前进行步骤**：`steps[0]` 通常已完成，用户更想看"当前在做什么"
- **无进度条**：只有 `n/m` 数字，缺乏直观的视觉进度反馈
- **步骤不可点击定位**：点击步骤名称无法跳转到 Timeline 对应位置
- **展开态步骤描述截断**：`truncate` 样式导致长步骤描述被裁剪，无法查看完整内容
- **只显示最新 Plan**：历史计划无法回溯（与会话详情页问题相同）

### 改进想法

- **折叠态显示当前进行中步骤**：优先展示 `status !== 'completed'` 的第一个步骤
- **步骤进度条**：在 `n/m` 旁边添加线性进度条，直观展示完成比例
- **步骤点击定位**：点击步骤后，主 Timeline 滚动到对应事件位置（需在步骤和 timeline item 之间建立关联）
- **步骤文字多行展开**：去掉 truncate，允许换行，或用 Tooltip 显示完整内容

## 7. 设置页（Settings）

**文件**: `src/components/manus-settings.tsx`

### 现状

- Dialog 形式，左侧导航 + 右侧内容
- 四个 Tab：通用配置（Agent）/ 模型提供商（LLM）/ A2A Agent / MCP 服务器
- 通用配置和 LLM 需要点击底部"保存"按钮；A2A 和 MCP 操作即时生效
- MCP 配置通过粘贴完整 JSON 添加
- 删除操作直接调 API，无确认步骤（乐观更新 + 失败回滚）

### 问题 / 痛点

- **保存逻辑不一致**：通用/LLM 需要手动保存，MCP/A2A 操作即时生效，容易混淆用户心智
- **api_key 无显示/隐藏切换**：`type="password"` 无法临时查看已填入的密钥是否正确
- **MCP JSON 编辑无结构验证**：粘贴 JSON 后只有 `JSON.parse` 的 SyntaxError 提示，无字段校验和格式化
- **删除操作无确认**：乐观删除虽然流畅，但误操作无补救机会
- **底部"保存"按钮对 MCP/A2A Tab 显示但无效**：容易让用户误以为 MCP 操作需要保存才生效
- **无连接测试功能**：LLM base_url 和 api_key 填好后无法验证是否可用
- **A2A 的 FieldDescription 复制了 MCP 的描述文案**（第 346-349 行），存在文案错误

### 改进想法

- **统一保存体验**：将通用/LLM 改为即时保存（blur 时自动保存），与 MCP/A2A 行为一致
- **api_key 显示切换**：在密码输入框右侧添加眼睛图标，切换 `type` 在 `password` 和 `text` 之间
- **MCP JSON 格式化与高亮**：引入 Monaco Editor 或简单的 JSON lint 提示替换纯 Textarea
- **删除二次确认**：点击删除图标后显示 Popover 确认框，而非直接执行
- **连接测试按钮**：在 LLM 配置区域添加"测试连接"按钮，调用后端接口验证配置有效性
- **配置导入/导出**：提供"导出所有配置"按钮，输出 JSON 文件，方便迁移和备份

## 8. 通用 UX 改进

### 暗色模式

当前样式大量使用 `text-gray-xxx`、`bg-white`、`bg-gray-50` 等硬编码颜色，暗色模式（`dark:` 前缀）几乎未实现。建议通过 CSS 变量统一管理颜色 token，然后逐页面适配。

### 空状态设计

多处空态只有一行文字（如"暂无 MCP 服务器"），可以增加插图、简短说明和引导操作按钮，提升空态引导体验。

### 移动端适配

会话详情页的双栏布局（主内容 + 右侧预览面板）在移动端无法使用。可以将预览面板改为从底部弹出的抽屉（`Sheet` 组件），适配小屏场景。

### 键盘无障碍

部分交互元素（如工具卡片、步骤列表项）未设置 `role="button"` 和 `tabIndex`，键盘用户无法触达，需补充 ARIA 属性。
